Date

Extracting year and month information from the cruise column and converting it to a convenient format. Creating base_date in the yyyy-mm-01 format to make sure it is treated as date in R. Used “cruise” column as reference rather than “date”, since some sampling is done early in the next month (e.g. sampling date is 7/1 but it’s from june cruise)

#create column with year and month
df <- raw_data %>%

  mutate(base_date = make_date(
      year  = 2000 + as.numeric(substr(cruise, 4, 5)),
      month = as.numeric(substr(cruise, 6, 7)),
      day   = 1))%>% 
 #move column 
  relocate(base_date, .after = date)

Temporal changes in krill size

Testing interannual variability using Linear Mixed Models because:

  • nested data (no need to collapse data to means)

  • not independent data (avoiding pseudoreplication and inflated significance)

  • considering uneven sampling effort

  • more suitable for different months and different numbers of cruises in year

In this model, year is used as fixed effect and cruise/jar as random factors, therefore testing if variability among years is larger than within-jar/cruise.

downsides: Some years have more cruises/samples than others, which can influence the precision Years are not considered directional, and this model will not show overall trends. Considering that the main goal is to identify changes in response to climate anomalies, this is not a problem.

Months

checking cruises per months:

cruise_per_month <- df %>%
    group_by(month = month(base_date)) %>%
  summarise(n_cruises = n_distinct(base_date))

Months of sampling represent seasons and have important data regarding seasonal variability. Months with sufficient amount of data will be used as random factors.

Station

checking stations of each line across the years, revealing multiple station per line, some N/A, some have “.”, some “TT”. conclusion: do not use station column as random factors

station_per_line <- df %>%
  group_by(line, station) %>%
  summarise(n_jars = n_distinct(jar), .groups = "drop")

Diel

estimating how many night cruises by listing cruises with multiple tows (>3 samples per line), don’t have time data yet. Since number of night samples is way lower than day, night samples will be excluded from the model.

night_cruises <- df %>%
  group_by(base_date, line) %>%
  summarise(n_jars = n_distinct(jar), .groups = "drop") %>%
  filter(n_jars > 3)

Using mid point size per row with length data only per type (juv/f/m) per jar per size range

df_lmm <- df %>%
  mutate(length_mid = (length_l + length_h) / 2) %>%
relocate(length_mid, .after = length_h)

Calculating one size per type per jar

df_lmm <- df_lmm %>%
  group_by(species_code, jar, type, base_date) %>%
  summarise(length_weighted = sum(length_mid * tot_num) / sum(tot_num),
    .groups = "drop")

excluding cruises from feb (n=1), apr (n=2), and oct (n=3)

df_lmm <- df_lmm %>%
  filter(!month(base_date) %in% c(2, 4, 10))
